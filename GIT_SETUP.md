# Git Setup Guide

## Files to Commit ✅

You should commit these files:

### Source Code
- `src/` - All source files (components, types, etc.)
- `demo/` - Demo application (optional, but useful for examples)

### Configuration Files
- `package.json` - Package configuration
- `vite.config.ts` - Vite build configuration
- `vite.demo.config.ts` - Demo build configuration
- `tsconfig.json` - TypeScript configuration
- `tsconfig.build.json` - Build TypeScript config
- `tsconfig.node.json` - Node TypeScript config
- `tailwind.config.js` - Tailwind configuration
- `postcss.config.js` - PostCSS configuration
- `index.html` - HTML entry point

### Documentation
- `README.md` - Main documentation
- `USAGE.md` - Usage guide
- `QUICK_START.md` - Quick start guide
- `PACKAGE_GUIDE.md` - Package guide
- `.npmignore` - npm ignore rules

### Git Files
- `.gitignore` - Git ignore rules

## Files NOT to Commit ❌

These are automatically excluded by `.gitignore`:
- `node_modules/` - Dependencies (install with `npm install`)
- `dist/` - Build output (generated by `npm run build`)
- `.DS_Store` - macOS system files
- `*.log` - Log files
- `.env*` - Environment variables

## Initial Commit Steps

```bash
# 1. Add all files (respects .gitignore)
git add .

# 2. Check what will be committed
git status

# 3. Commit
git commit -m "Initial commit: Vue3 Spin Wheel component"

# 4. Push to remote (if you have one)
git push origin master
```

## Recommended Commit Message Format

```
feat: Initial commit - Vue3 Spin Wheel component

- Add SpinWheel component with smooth animations
- Add API integration support
- Add demo application
- Add TypeScript support
- Add build configuration for npm package
```

## What Gets Published to npm

When you run `npm publish`, only these are included (based on `package.json` files field):
- `dist/` folder (after build)
- Files listed in `package.json` "files" field

The `demo/` folder is excluded from npm package (via `.npmignore`), but it's good to keep in git for documentation.
